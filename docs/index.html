Architecture

How it works

Simple and transparent. No hidden APIs, no cloud services, no telemetry.

1
opsmate reads config from ~/.opsmate/config.yaml
2
Generates .mcp.json pointing to opsmate mcp <server> subcommands
3
Injects DevOps system prompt via CLAUDE.md
4
Launches Claude Code — which spawns MCP servers as needed
5
Each MCP server communicates over stdio — no ports, no API keys
6
On exit — clean up .mcp.json and CLAUDE.md markers
Security

Safety first

Built for production environments with multiple layers of protection.

👁

Read-only mode

--readonly disables apply, scale, delete, exec

✅

Confirmation prompts

Destructive operations require explicit approval

🔐

Secret redaction

Passwords, tokens, and keys masked in output

📦

Namespace restrictions

Limit K8s access to specific namespaces

📜

Log limits

Prevents OOM from massive log outputs

🔌

No network exposure

MCP servers use stdio, not HTTP
